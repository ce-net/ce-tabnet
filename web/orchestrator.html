End-to-end tok/s is measured from token arrival times in this console — it is the honest product number. Per-stage compute comes from each tab's metrics telemetry when the coordinator forwards it; otherwise it shows —. Single-stream latency carries a pipeline-depth tax (a token traverses every stage); aggregate throughput across concurrent prompts is the win. See docs/scaling-to-300b.md.

ce-tabnet — pipeline-parallel LLM inference across browser tabs. Honest math: a tab holds ~0.5–2 GB of WebGPU weights, so 300B @ 4-bit (~150 GB) needs ~100–300 tabs. The same code path runs a small model on a few tabs today and a 300B-class model on hundreds. Coordination: Cloudflare Worker + Durable Object (one per run).
